# AddExercise slice review

## SUMMARY

The AddExercise slice lets an authenticated trainer add an exercise to one of the trainer’s workout plans. It validates the exercise fields, verifies plan ownership, persists the exercise, and returns the created exercise with `201 Created`. [src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseEndpoint.cs:8-25](../src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseEndpoint.cs#L8-L25); [src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseHandler.cs:10-40](../src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseHandler.cs#L10-L40)

## 1. Responsibility & business requirement

The business capability is adding an exercise to a client’s existing workout plan. The route is `/api/workout-plans/{planId:guid}/exercises`; the route group supplies `planId`, and the endpoint accepts `AddExerciseRequest`, invokes the handler, and returns the created exercise location and representation. [src/PersonalTrainer.Api/Features/WorkoutPlans/WorkoutPlansFeatureExtensions.cs:23-30](../src/PersonalTrainer.Api/Features/WorkoutPlans/WorkoutPlansFeatureExtensions.cs#L23-L30); [src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseEndpoint.cs:8-25](../src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseEndpoint.cs#L8-L25)

`AddExerciseRequest` contains `Name`, `Sets`, `Reps`, optional `WeightKg`, optional `Notes`, `OrderIndex`, and optional `Tags`. ## 6. Main risks (2)

1. **Ownership predicate:** Keep `w.Id == planId && w.TrainerId == trainer.Id` together. Removing the trainer condition would allow a trainer to target another trainer’s plan through this handler. [src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseHandler.cs:12-22](../src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseHandler.cs#L12-L22)
2. **Endpoint-bound validation:** Basic validation is installed on the HTTP endpoint, while the handler directly maps accepted values and does not visibly enforce higher-level ordering or tag rules. Changes that add another call path or business rule need corresponding enforcement and tests. [src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseEndpoint.cs:8-14](../src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseEndpoint.cs#L8-L14); [src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseHandler.cs:24-38](../src/PersonalTrainer.Api/Features/WorkoutPlans/AddExercise/AddExerciseHandler.cs#L24-L38)

## CLOSING SUMMARY

Before changing this slice, preserve the ownership check that binds the route’s `planId` to the current trainer. Also remember that the validator handles field-level constraints, but the handler directly persists the accepted request and does not visibly enforce higher-level ordering or tag rules.
